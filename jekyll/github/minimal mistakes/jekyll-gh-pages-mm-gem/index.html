    Howto: Create a blog with Jekyll, Github pages, Minimal Mistakes gems

    less than 1 minute read

    Trying to build a blog. Clone repo unbelievably messy. Gem seems like a good idea undocumented…

    # Clone and enter repo
git clone git@github.com:${USER}/${REPONAME}.git
cd ${REPONAME}

# Create gh-pages branch.
# Github seems to enable the gh-pages automatically
# if repo has this branch
git branch gh-pages
git checkout gh-pages
git push --set-upstream origin gh-pages

# Clone repo again within the repo and move it to
# _site folder (that's where Jekyll will generate)
# the webpage
git clone git@github.com:${USER}/${REPONAME}.git
mv ${REPONAME} _site
cd _site
git checkout gh-pages
cd ..
echo _site > .gitignore
